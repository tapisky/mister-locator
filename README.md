# 🎯 mister-locator

> Extract robust, resilient [Playwright](https://playwright.dev) locators from any web page — as a CLI tool or Chrome Extension.

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

---

## Why mister-locator?

Playwright's built-in `codegen` only captures locators during recorded interactions. **mister-locator** lets you:

- **Point at any element** on any live page and instantly get the best possible Playwright locator
- **Scan an entire page** and generate a typed locator map ready to paste into your tests
- Follow **Playwright's recommended locator priority** (`getByRole` → `getByLabel` → ... → CSS → XPath) automatically

---

## Packages

| Package | Description |
|---|---|
| `@mister-locator/core` | Locator engine — pure TS, no DOM/Node deps |
| `@mister-locator/cli` | Terminal tool — scan any URL via headless Playwright |
| `@mister-locator/extension` | Chrome Extension — click to pick any element live |

## Getting started

### Prerequisites

- Node.js 20+
- pnpm 9+

```bash
# Clone the repo
git clone https://github.com/your-username/mister-locator.git
cd mister-locator

# Install pnpm
npm install -g pnpm

# Install all dependencies
pnpm install

# Install Playwright browsers (for the CLI)
pnpm --filter @mister-locator/cli exec playwright install chromium

# Build everything
pnpm build
```

## CLI usage

```bash
# Scan all interactive elements on a page (table output)
node packages/cli/dist/index.js scan https://example.com

# Output as JSON
node packages/cli/dist/index.js scan https://example.com --format json

# Generate a TypeScript locator file
node packages/cli/dist/index.js scan https://example.com --format code --output locators.ts

# Show the browser window while scanning
node packages/cli/dist/index.js scan https://example.com --no-headless
```

### Example output (`--format table`)

```
<button> [getByRole] 95%
  page.getByRole('button', { name: 'Sign in' })

<input> [getByLabel] 90%
  page.getByLabel('Email address')
  alternatives: 2

<a> [getByRole] 95%
  page.getByRole('link', { name: 'Forgot password?' })
```

### Example output (`--format code`)

```ts
// locators.ts — generated by mister-locator
import { Page } from '@playwright/test';

export function getLocators(page: Page) {
  return {
    /** strategy: getByRole | confidence: 95% */
    signIn: page.getByRole('button', { name: 'Sign in' }),
    /** strategy: getByLabel | confidence: 90% */
    emailAddress: page.getByLabel('Email address'),
  };
}
```

## Chrome Extension

1. Run `pnpm --filter @mister-locator/extension build`
2. Open Chrome → `chrome://extensions`
3. Enable **Developer mode**
4. Click **Load unpacked** → select `packages/extension/`
5. Click the 🎯 icon in the toolbar
6. Hit **"Pick element"** and click anything on the page

The extension shows the best locator with confidence score, all alternatives, and a one-click copy button.

## Locator strategy priority

| Strategy | Confidence | When used |
|---|---|---|
| `getByRole` | 95% | Element has an ARIA role + accessible name |
| `getByLabel` | 90% | Form control has an associated `<label>` |
| `getByPlaceholder` | 85% | Input has a `placeholder` attribute |
| `getByTestId` | 88% | Element has `data-testid` / `data-cy` / `data-qa` |
| `getByAltText` | 80% | Image has `alt` text |
| `getByTitle` | 75% | Element has a `title` attribute |
| `getByText` | 70% | Element has short, static visible text |
| CSS | 30–60% | Fallback — id-based or full path |
| XPath | 20% | Last resort |
